
# 一、数据的表示
## 考点
1. 进制转换
2. 码制（源码、反码、补码、移码）
3. 浮点数的表示
4. 逻辑运算

## 1. 进制转换
### 1.1 计算机中的基本进制：
* 十进制（无任何说明时，数字均为10进制。尾号为D）
* 二进制（B）
* 十六进制（H）
数码、基数等

### 1.2 数据表示：按权展开（转十进制）
$S= \sum_{i=0}^n a_n*b^n$，其中，$a_n$表示的是b进制数上倒数第$n-1$位上的数据

### 1.3 短除法，十进制转其他进制

使用除基取余法将十进制转为R进制。商为0为止。余数从下往上记录。

### 1.4 数据表示：减法。（这可以用来凑整数）
不常用

### 1.5 进制转换——二、八、十六进制数转换
通过划分比特位进行数据转换。不足对高位补0.

## 2.码制（源码、反码、补码、移码）
- 原码：最高位是符号位，其余地位表示数值绝对值
- 反码：正数的反码与原码相同，负数的反码是其绝对值按位取反（符号位不变）
- 补码：正数的补码与原码相同，负数的补码是其反码末位加一，符号位不变
- 移码：与补码的符号位按位取反。（用于浮点数）

定点整数一般用补码表示。

原码表示$-(2^{(n-1)}-1)$~$+(2^{n-1})$

定点小数一般用移码表示。

## 3.浮点数的表示

浮点数的表示：N = $尾数*基数^{指数}$

其中，尾数为定点小数，基数为2，指数不固定。指数为阶码，定点整数

运算过程：对阶 > 尾数计算 > 结果格式化

尾数用补码，阶码用移码。（IEEE 754中，尾数也可以用原码表示）

阶码的尾数决定数的表示范围，
尾数的位数表示数的有效精度。
对阶时，小数向大数看齐。
对阶通过小数尾数右移实现

阶符：阶码符号位

数符：尾数的符号位



## 4. 关系运算符

### 关系运算符优先级：
1. < <= > >= 优先级相同（高）

2.    ==  != 优先级相同（低）

3. 关系运算符的优先级低于算数运算符
4. 关系运算符的优先级高于赋值运算符

### 逻辑运算
逻辑变量之间的运算称之为逻辑运算。二进制数1和0可以代表真假

优先级：
* 或
* 与
* 异或
* 非 

真值表：描述一个逻辑表达式与其变量之间的关系

&& || ! 三个符号

总体优先级：! > 算术运算符>关系运算>&&>||>赋值运算符

**短路原则**按顺序来，能出结果的时候，后面不去判断。
